# Resume Rewrite Rules

Use these rules when tailoring the resume to a JD.

## Truthfulness Rules

- Do not invent metrics. Use `[补充真实数据]` when a number is needed.
- Do not turn exposure into ownership. "参与" cannot become "主导" unless evidence supports it.
- Do not add tools, companies, titles, certificates, publications, or degrees not provided.
- If a claim is plausible but not evidenced, mark it as `需要确认`.

## Rewrite Priority

Rewrite in this order:

1. Title / target role line
2. Professional summary
3. Core skills
4. Most relevant project or experience bullets
5. Less relevant experience ordering
6. Optional portfolio or proof links

## Bullet Formula

Prefer one of these formats:

- `Action + object + method/tool + measurable result`
- `Problem + action + outcome`
- `Scope + responsibility + impact`
- `Built/led/analyzed/optimized + what + using what + why it mattered`

Good bullet:

```text
Built a JD-to-resume matching workflow that extracts role requirements, scores candidate fit, and generates targeted rewrite suggestions, reducing manual screening time by [补充真实数据].
```

Weak bullet:

```text
Responsible for resume optimization and job matching.
```

## ATS Keyword Handling

- Map JD keywords to resume evidence instead of stuffing keywords.
- Put hard-skill keywords in `Skills` only if the resume contains supporting experience.
- Mirror role language when truthful. Example: if JD says `stakeholder management`, rewrite `跨部门沟通` as `stakeholder coordination` only if context supports it.
- Keep repeated keywords natural. One strong bullet is better than five awkward mentions.

## Summary Rewrite

Keep summary to 2-4 lines.

Structure:

1. Role identity and years/scope if known
2. Most relevant domain/tool strength
3. Evidence of impact
4. Target role alignment

If evidence is thin, avoid overclaiming. Use a modest but targeted summary.

## Project Rewrite

For each important project:

- Start with the problem or business context.
- Name the user's actual contribution.
- Bring relevant tools and methods closer to the front.
- End with measurable or observable outcome.
- If outcome is missing, ask the user for data before final resume use.

## Output Style

- Be direct and practical.
- Show before/after when original text is available.
- Explain why each rewrite improves match.
- Flag anything the user must verify before using publicly.

## Red Flags

Avoid these:

- Generic "highly motivated" summaries.
- Keyword lists not backed by experience.
- Inflated seniority.
- Overly polished bullets that sound disconnected from the actual resume.
- Advice that tells the user to apply everywhere.
